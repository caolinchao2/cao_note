严格模式 ​

React.StrictMode：

严格模式只在开发模式下运行，不会与生产模式冲突，可以在任何地方使用，如同Fragment；

Strict mode有助于：

  • 识别具有不安全生命周期的组件
  • 有关旧式字符串ref用法的警告（使用第三方库很难确保不使用这些生命周期的方法，加上这个可以帮忙判断）
  • 关于已弃用的findDOMNode用法的警告（findDOMNode是传入一个ref则可以获取真实dom元素）
  • 检测意外的副作用
  • 检测遗留的context API

错误边界处理 ​

（1）创建一个文件ErrorBoundary.js，写入以下内容

jsx
export default class ErrorBoundary extends React.Component {
    state = {
        hasError: false,
        error: null,
        errorInfo: null
    }
    // 子元素发生错误时执行该事件
    componentDidCatch(error,errorInfo){
        this.setState({
            hasError: true,
            error,
            errorInfo
        })
    }
    render() {
        if(this.state.hasError){
            // 出错渲染render方法,render在使用该组件时定义
            return <div>{this.props.render(this.state.error,this.state.errorInfo)}</div>
        }
        // 没有出错则渲染子元素, 当子元素出错会触发compnentDidCatch
        return this.props.children
    }
}

类定义组件 ​

PureComponent (类定义组件只有props，没有state) ​

Component 是state|props变了，默认是不做对比，直接更新

PureComponent是是state|props变了，默认都要做对比的，对比完之后，才决定是否更新。

​ PureComponent做的是浅比较,如果父组件传递的数据是引用类型（数组|对象），那么你父组件需要通过拷贝赋值，才能引起子组件的更新。

jsx
import React, { PureComponent } from "react";

export default class Banner extends PureComponent {
 
  render() {
    console.log("----banner render ----");
    return (
      <div className="box">
        <h3>轮播-（props）</h3>
        <div>banner:{this.props.banner}</div>
        <div>arr:{JSON.stringify(this.props.arr)}</div>
      </div>
    );
  }
}

shouldComponentUpdate (类定义组件有state，有props) ​

1.子组件有自己的state和props，更新期可能是state或者props引起的； 2.为了跳过某次更新，需要在shouldComponentUpdate中做判断，但是需要知道是props引起的还是state引起的； 3.初始化一个标记isPropsChange，用来判断props是否发生了改变。默认是false,如果props变了，就会进入UNSAFE_componentWillReceiveProps，改为true.更新完成后改为false. 4.在shouldComponentUpdate中查看isPropsChange，true，就是props变了，开始对比上一次和这一次的值；如果是false，代表state变了，直接更新。

jsx
import React, { Component } from "react";
export default class Cate extends Component {
  constructor() {
    super();
    this.state = {
      num: 10,
    };
    // 初始化一个变量，来标记是否是props变了，true就是props变了，false就不是
    this.isPropsChange = false;
  }
  UNSAFE_componentWillReceiveProps() {
    this.isPropsChange = true;
  }
  shouldComponentUpdate(nextProps, nextState) {
    //   更新期有可能是props引起的，也可能是state引起的。
    // 如果是props引起的，那么需要查看两次的cate是否一致，一致，就不更新，不一致，就更新；
    // 如果state引起的，就不用做判断，直接更新

    if (this.isPropsChange) {
      if (nextProps.cate === this.props.cate) {
        return false;
      } else {
        return true;
      }
    }

    return true;
  }
  render() {
    console.log("子组件渲染了");
    return (
      <div className="box">
        <h3>分类子组件-类定义写法--UNSAFE_componentWillReceiveProps+shouldComponentUpdate+componentDidUpdate 【state+props】</h3>
        <div>cate:{this.props.cate}</div>
        <div>state.num:{this.state.num}</div>
        <button onClick={() => this.setState({ num: this.state.num + 1 })}>
          num++
        </button>
      </div>
    );
  }
  componentDidUpdate() {
    this.isPropsChange = false;
  }
}
import React, { Component } from "react";
export default class Cate extends Component {
  constructor() {
    super();
    this.state = {
      num: 10,
    };
    // 初始化一个变量，来标记是否是props变了，true就是props变了，false就不是
    this.isPropsChange = false;
  }
  UNSAFE_componentWillReceiveProps() {
    this.isPropsChange = true;
  }
  shouldComponentUpdate(nextProps, nextState) {
    //   更新期有可能是props引起的，也可能是state引起的。
    // 如果是props引起的，那么需要查看两次的cate是否一致，一致，就不更新，不一致，就更新；
    // 如果state引起的，就不用做判断，直接更新

    if (this.isPropsChange) {
      if (nextProps.cate === this.props.cate) {
        return false;
      } else {
        return true;
      }
    }

    return true;
  }

    return (
      <div className="box">
        <h3>分类子组件-类定义写法--UNSAFE_componentWillReceiveProps+shouldComponentUpdate+componentDidUpdate 【state+props】</h3>

}

函数组件 React.memo() 【高阶组件】 ​

React.memo(组件) 作用：让函数组件减少渲染次数，只有自己依赖的数据变了，才会重新渲染。 React.memo()也是做得浅比较。,如果父组件传递的数据是引用类型（数组|对象），那么你父组件需要通过拷贝赋值，才能引起子组件的更新。

jsx
import React from 'react'

function Goods(props) {
    console.log("goods开始计算了");

    </div>
  )
}
export default React.memo(Goods)

import React from "react";

//销毁之前
  componentWillUnmount(){

componentDidCatch 错误边界处理 ​

react中，默认情况下，是一旦有错误，整个页面都空白了。但是我们希望某一个组件有错误，只有这个组件加载失败，其他组件正常展示。此刻我们封装一个 ErrorBounday组件来实现。 ComponentDidCatch是组件自己的Children出错了，才会触发。

所以我们初始化一个haveError=false,一旦出错，变为true.根据haveError来条件渲染页面。

jsx
import React, { Component } from "react";

export default class ErrorBoundary extends Component {
  constructor() {
    super();
    this.state = {
      haveError: false,
    };
  }
  componentDidCatch() {
    console.log("===错误了！！！！！======");
    this.setState({
      haveError: true,
    });
  }
  render() {
    let { haveError } = this.state;
    return (
      <>{haveError ? <div>此处加载失败</div> : this.props.children}</>
    );
  }
}

解决内存泄露 ​

Can't perform a React state update on an unmounted component. This is a no-op, but it indicates a memory leak in your application. To fix, cancel all subscriptions and asynchronous tasks in the componentWillUnmount method 报红

原因：在willMount中有setState的事件，

jsx
class Test extends React.Component {
    _isMounted = false
    componentDidMount(){
        this._isMounted = true
        this._isMounted && this.setState({...})
    }
    componentWillUnmount(){
        this._isMounted = false
    }
}

}
